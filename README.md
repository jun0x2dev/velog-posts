# Velog Workspace

이 저장소는 jun0x2dev의 Velog 게시글 작성을 위한 워크스페이스입니다.
다양한 IT 기술 스택, 개인 프로젝트 경험, 그리고 학술적인 탐구를 기록하고 공유하기 위한 목적으로 구성되었습니다.

## 디렉토리 구조 및 시리즈 설명

-   **`posts/`**: Velog에 게시될 모든 글이 시리즈별로 분류되어 저장되는 핵심 디렉토리입니다.
    -   **`01_CS_Algorithm/`**: 자료구조, 알고리즘 문제 풀이, 시간 복잡도 분석 등을 다룹니다.
    -   **`02_Back-end/`**: Java, Spring Boot, API 설계 등 백엔드 기술 관련 내용을 기록합니다.
    -   **`03_Front-end/`**: React, HTML/CSS, UI/UX 개선 경험 등 프론트엔드 기술을 공유합니다.
    -   **`04_Database/`**: RDB (MySQL 등) 설계, 쿼리 최적화, DB 개념 등을 정리합니다.
    -   **`05_Infra/`**: AWS, Docker, 배포 환경 구축 및 네트워크 지식 등을 다룹니다.
    -   **`06_AI/`**: 인공지능 기초 이론, 머신러닝/딥러닝 모델 학습 기록 등을 포함합니다.
    -   **`07_Tech_Journal/`**: 최신 IT 트렌드, 새로운 기술 스택 경험 및 리뷰를 작성합니다.
    -   **`08_Graduate_School/`**: 대학원 준비 과정, 수업 리뷰, 논문 요약 등 학술적 내용을 기록합니다.
    -   **`09_Personal_Project/`**: 개인 프로젝트의 기획, 트러블슈팅, 회고록 등을 다룹니다.
    -   **`draft/`**: 발행 전 아이디어 스케치 및 임시 메모, 미완성 글을 보관하는 공간입니다.

-   **`images/`**: 게시글에 사용될 이미지 파일들을 저장하는 디렉토리입니다. 글 내용과 관련된 이미지를 이곳에 관리합니다.

## Velog 게시글 작성 규칙

효율적이고 일관된 게시글 작성을 위해 다음 규칙을 준수해 주세요.

1.  **파일 및 폴더명 규칙:**
    -   각 게시글 파일은 해당 시리즈 폴더(예: `posts/01_CS_Algorithm/`) 내에 위치해야 합니다.
    -   파일 이름은 `YYYYMMDD-게시글_제목.md` 형식으로 작성합니다. (예: `20240208-Java_Stream_API_정리.md`)
    -   띄어쓰기는 언더스코어(`_`)로 대체하거나 하이픈(`-`)을 사용할 수 있습니다.

2.  **게시글 구조:**
    -   모든 게시글은 일관된 흐름을 위해 다음 구성을 따르는 것을 권장합니다.
        -   `# [게시글 제목]`: 최상위 제목은 하나만 사용합니다.
        -   `## 서론`: 게시글의 배경, 목적, 다룰 내용 등을 간략하게 소개합니다.
        -   `## 본론`: 주제에 대한 상세한 설명, 코드 예시, 개념 정리 등을 포함합니다. (하위 `###` 제목 활용)
        -   `## 결론 및 요약 / 회고`: 배운 점, 느낀 점, 앞으로의 방향 등을 정리합니다.
        -   `## 참고 자료`: 인용한 문서, 링크 등을 명시합니다.

3.  **마크다운 문법 활용:**
    -   코드 블록은 백틱(` `` `) 3개로 감싸고 언어 종류를 명시하여 신택스 하이라이팅을 적용합니다. (예: ```java, ```javascript)
    -   중요한 내용은 볼드체(`**내용**`), 강조할 내용은 이탤릭체(`*내용*`)를 사용합니다.
    -   적절한 헤딩(`##`, `###`, `####`)을 사용하여 문서 구조를 명확히 합니다.
    -   목록(`-` 또는 `1.`)을 활용하여 가독성을 높입니다.

4.  **이미지 사용:**
    -   게시글에 사용될 이미지는 `images/` 디렉토리에 저장하고, 마크다운에서 상대 경로로 참조합니다.
    -   예시: `![이미지 설명](/images/series_name/image_file_name.png)`
    -   이미지 파일명도 `시리즈명_게시글명_순번.png`와 같이 체계적으로 관리하는 것을 권장합니다.

5.  **Velog 태그 (Frontmatter):**
    -   각 게시글 파일 상단에 Velog의 Frontmatter 형식을 사용하여 태그(`tags:`)를 명시합니다.
    -   예시:
        ```markdown
        ---
        title: "게시글 제목"
        description: "게시글 요약 설명"
        date: YYYY-MM-DD
        update: YYYY-MM-DD
        tags: [tag1, tag2, tag3]
        ---
        ```

6.  **작성 흐름:**
    -   새로운 아이디어나 미완성 글은 `draft/` 폴더에서 먼저 작성합니다.
    -   어느 정도 내용이 완성되면 해당 시리즈 폴더로 이동하여 최종 다듬기 작업을 진행합니다.
    -   게시 전에는 오탈자 및 내용의 정확성을 다시 한번 검토합니다.

이 `README.md`는 Velog 포스팅을 위한 중요한 가이드라인이 될 것입니다.
