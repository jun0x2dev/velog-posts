# 02_Back-end

## 시리즈 목적
백엔드 기술 스택 (Java, Spring Boot 등) 학습 및 프로젝트 구현 기록을 위한 공간입니다.

## 학습 로드맵
### Phase 1: Deep into Java
1. **JVM과 메모리 구조** - 자바가 돌아가는 근본적인 원리
2. **자바의 정석: 객체지향(OOP)** - 인터페이스와 추상화의 실무적 활용
3. **자바 8 이후의 변화** - 람다, 스트림, 그리고 Optional
4. **멀티 스레드와 동시성** - 백엔드 성능의 핵심

### Phase 2: Spring Boot Magic (Core & Data)
5. **IoC와 DI** - 제어의 역전, 스프링이 대신 해주는 것들
6. **AOP와 프록시** - 반복되는 코드를 마법처럼 제거하기
7. **JPA와 영속성 컨텍스트** - DB를 객체처럼 다루는 기술
8. **Spring Data JPA 실무** - QueryDSL과 성능 최적화 전략

### Phase 3: Building Real World Apps
9. **Spring MVC 구조와 Flow** - 요청부터 응답까지의 여정
10. **예외 처리와 유효성 검사** - 견고한 API 설계를 위한 @ControllerAdvice
11. **Spring Security와 OAuth2** - 보안의 기초부터 소셜 로그인까지
12. **테스트 코드의 정석** - JUnit5와 Mockito를 활용한 단위/통합 테스트
13. **외부 API 연동** - RestTemplate과 WebClient 활용법
14. **CI/CD와 배포** - Docker와 Github Actions를 활용한 자동화 환경 구축

### Phase 4: Advanced & Reactive Stack
15. **리액티브 프로그래밍과 Spring WebFlux** - 비동기 논블로킹 패러다임의 이해
16. **성능 모니터링과 프로파일링** - 프로메테우스와 그라파나 활용하기
17. **MSA 아키텍처 기초** - 서비스 분리와 API Gateway의 역할

## 작성 가이드
- 이론적인 내용뿐만 아니라 실무에서 겪은 트러블슈팅 경험을 포함합니다.
- 복잡한 아키텍처는 시각화된 자료를 활용하여 설명합니다.
- 특정 기술 (예: Spring Security, JPA) 학습 내용을 정리합니다.
- API 설계 과정, 비즈니스 로직 구현 방법, 트러블슈팅 경험을 공유합니다.
