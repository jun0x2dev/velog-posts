# 04_Database

## 시리즈 목적
데이터베이스 (RDB, NoSQL 등) 설계, 쿼리 최적화, 개념 정리 기록을 위한 공간입니다.

## 학습 로드맵
1. **DB 설계의 정석** - ERD 모델링과 정규화(Normalization)의 이해 (완료)
2. **SQL의 심장 인덱스(Index)** - B-Tree 구조와 쿼리 성능 최적화 전략
3. **트랜잭션과 격리 수준** - ACID 원칙과 데이터 무결성
4. **MySQL 깊이 파헤치기** - InnoDB 엔진 아키텍처와 성능 최적화 포인트
5. **PostgreSQL 깊이 파헤치기** - 객체-관계형 DB의 강력함과 MVCC의 이해
6. **RDB vs NoSQL: 기술 선택의 기준** - 서비스 성격에 따른 아키텍처 설계 전략
7. **Redis의 정석** - 인메모리 데이터 저장소의 활용 패턴과 분산 락
8. **SQL 튜닝 실전** - 실행 계획 분석 및 병목 지점 개선

## 작성 가이드
- ERD 설계, 정규화/비정규화 전략을 설명합니다.
- SQL 쿼리 작성 팁, 인덱스 활용, 쿼리 성능 최적화 방법을 다룹니다.
- 트랜잭션, 동시성 제어 등 데이터베이스 핵심 개념을 정리합니다.
