# 기초 자료구조 이해: 스택(Stack)과 큐(Queue)의 개념 및 실무 활용

## 서론
안녕하세요! 지난번 '시간 복잡도' 편에서 우리 본체를 지키는 법을 배웠다면, 오늘은 데이터를 어떻게 쌓고 빼는지에 대한 **'정리의 기술'**을 배워보려 합니다.

코딩을 하다 보면 데이터를 잠시 어딘가에 보관했다가 꺼내 써야 할 때가 정말 많죠. 그런데 아무렇게나 던져두면 나중에 찾기 힘들어집니다. 그래서 우리는 **스택(Stack)**과 **큐(Queue)**라는 아주 훌륭한 정리 도구를 사용합니다. 이름은 거창하지만 알고 보면 우리 일상속에 늘 있는 녀석들이에요!

## 본론

### 1. 스택(Stack): "나중에 온 놈이 먼저 나간다!" (LIFO)
스택은 말 그대로 '쌓아 올리는' 구조입니다. 가장 좋은 비유는 바로 **프링글스 통**이에요.

![프링글스 스택 비유 짤](/images/01_CS_Algorithm/stack_queue/pringles_stack.png)
*(여기에 '프링글스 통에서 감자칩을 꺼내는' 이미지나 '쌓여있는 접시' 짤을 넣어주세요!)*

제일 먼저 넣은 감자칩은 통 맨 밑에 깔려서 가장 나중에 나오게 되죠. 이걸 전문 용어로 **LIFO (Last In, First Out)**라고 부릅니다.

#### 🛠️ 스택의 주요 연산
- **Push:** 데이터를 통에 집어넣기
- **Pop:** 가장 위에 있는 데이터를 꺼내기 (냠냠)

#### 💻 실전 예시: 뒤로 가기와 실행 취소 (Undo)
우리가 브라우저에서 '뒤로 가기'를 누르거나, 코딩하다가 `Ctrl + Z`를 누를 때 스택이 열일하고 있는 겁니다. 가장 최근에 했던 행동을 가장 먼저 되돌려야 하니까요!

```java
Stack<String> undoStack = new Stack<>();
undoStack.push("글쓰기");
undoStack.push("글꼴 변경");
undoStack.push("색상 변경");

System.out.println(undoStack.pop()); // 결과: 색상 변경 (가장 최근 작업이 취소됨)
```
> **컴퓨터의 평:** "방금 한 실수는 내가 기억하고 있을게. 언제든 Pop만 해!"

---

### 2. 큐(Queue): "먼저 온 놈이 먼저 나간다!" (FIFO)
큐는 '줄 서기'입니다. 맛집 앞에서 기다리는 선량한 시민들의 모습 그 자체죠.

![맛집 줄 서기 큐 비유 짤](/images/01_CS_Algorithm/stack_queue/queue_line.png)
*(여기에 '맛집 앞에 길게 늘어선 줄'이나 '은행 번호표' 짤을 넣어주세요!)*

먼저 온 사람이 먼저 들어가서 밥을 먹는 평화로운 구조, 이걸 **FIFO (First In, First Out)**라고 합니다.

#### 🛠️ 큐의 주요 연산
- **Enqueue:** 줄 뒤에 서기 (입장 대기)
- **Dequeue:** 줄 맨 앞사람이 입장하기 (식사 시작)

#### 💻 실전 예시: 프린터와 마우스 클릭
프린터에 인쇄 명령을 10번 내리면, 당연히 첫 번째로 누른 문서가 먼저 나와야겠죠? 이런 **순서가 중요한 작업**에는 무조건 큐가 등장합니다.

```java
Queue<String> printQueue = new LinkedList<>();
printQueue.add("이력서.pdf");
printQueue.add("포트폴리오.pdf");

System.out.println(printQueue.poll()); // 결과: 이력서.pdf (먼저 요청한 게 먼저 나옴)
```
> **컴퓨터의 평:** "새치기는 금물! 차례대로 처리해 줄게."

---

### 3. 스택 vs 큐, 한눈에 비교하기

| 특징 | 스택 (Stack) | 큐 (Queue) |
| :--- | :--- | :--- |
| **핵심 원리** | **LIFO** (후입선출) | **FIFO** (선입선출) |
| **일상 비유** | 프링글스, 쌓인 접시 | 맛집 줄 서기, 고속도로 톨게이트 |
| **주요 사용처** | 실행 취소(Undo), 재귀 함수, DFS | 프로세스 관리, 인쇄 대기열, BFS |
| **데이터 추가** | Push | Enqueue (add) |
| **데이터 삭제** | Pop | Dequeue (poll) |

### 4. 주의할 점: "넘치거나 비었거나"
- **Stack Overflow:** 스택이 꽉 찼는데 더 넣으려고 할 때 발생합니다. (재귀 함수 무한 루프 돌리면 자주 만나는 그 유명한 사이트 이름 맞습니다!)
- **Empty Exception:** 비어있는 스택이나 큐에서 데이터를 꺼내려고 하면 컴퓨터가 당황해서 에러를 뿜어내니 항상 확인하는 습관을 들입시다.

## 결론 및 요약 / 회고
오늘은 데이터를 정리하는 가장 기초적인 두 가지 방법, 스택과 큐를 알아봤습니다.
1. **거꾸로** 되돌려야 한다면? → **스택!**
2. **순서대로** 처리해야 한다면? → **큐!**

이 두 가지만 제대로 알아도 앞으로 배울 복잡한 알고리즘(DFS, BFS 등)의 절반은 먹고 들어가는 겁니다. 자료구조, 생각보다 별거 아니죠?
